# 배열(array)

: 탐색 정보가 저장되어 있는 자료구조
![[Pasted image 20220816152754.png]]
(a)처럼 개별적인 이름을 가지며 다양한 위치에 저장된 경우 만약 탐색 대상 원소들의 개수가 수십 만 개 수준으로 증가하면 알고리즘 설계 및 구현에서 번거로운 문제가 발생한다.

개수가 클 경우 (b)에서 보는 것과 같이 배열(array)를 구성하여 사용하는 것이 일반적이다. 배열은 동일한 자료형의 원소(element)들을 컴퓨터 메모리의 연속적인 주소 영역에 저장하며, 원소가 저장된 위치를 순서에 따라 가리키는 인덱스(index)를 사용해 지정된 원소를 읽어오거나 새로운 값으로 변경할 수 있다.



# 순차 탐색 (sequential search)

: 배열에 저장되어 있는 문자들을 처음부터 차례대로 비교하며 검색대상 문자인가를 확인하고, 만약 검색대상 문자가 발견된 경우 그 위치(배열의 인덱스)를 반환

![[Pasted image 20220816152822.png]]

탐색 대상 데이터 **개수가 많지 않고** **수시로 새로운 데이터가 추가**되고 **기존 데이터가 삭제**되어야 하는 상황에서 데이터 탐색 빈도수가 높지 않을 경우에는 순차탐색 기능을 사용할 수 있다.

다만, 탐색 대상 데이터의 개수가 매우 큰 상황에서 순차 탐색 방법을 사용하는 경우 위의 그림에서처럼 N을 찾는 경우는 매우 신속하게 결과를 얻을 수 있지만, Q를 찾는 경우는 거의 모든 단어를 다 찾아본 후에야 결과를 얻을 수 있기 때문에 매우 오래 걸리게 된다.

**따라서 탐색 대상 데이터 규모가 크고 탐색 회수가 빈번할 경우에는 순차탐색이 좋은 방안이 되지 않는다.**



# 이진 탐색 (binary search)

이진 탐색은 탐색 대상 데이터들이 정렬되어 있는 상황에서 사용할 수 있다.
![[Pasted image 20220816152848.png]]
그림과 같이 문자들이 문자코드 크기 순서대로 정렬되어 있는 상태에서 탐색 대상 키 문자가 주어지면 이를 배열의 중간값 (그림의 경우 mid 인덱스가 가리키는 ‘M’ 문자 값)과 비교하여 탐색 키의 값이 중간값과 동일하면 그 mid 값을 반환한다. 만약 탐색 키 값이 mid 인덱스 위치의 중간값보다 작으면 0~mid-1 인덱스 구간을 이진 탐색 방법으로 탐색하며, 만약 탐색 키 값이 중간값 보다 크면 mid +~~ N-1 구간을 이진 탐색 방법으로 탐색한다.

이진 탐색 알고리즘에서는 탐색의 단계가 진행될 때 마다 탐색 대상 구간이 절반씩 줄어들기 때문에 전체 탐색 대상 배열이 N개의 원소를 가질 때 대략 log2(N)번의 이진 탐색이 진행되면 탐색 결과를 확정지을 수 있어 탐색 대상 배열의 규모가 클 경우 순차 탐색보다 훨씬 빠른 탐색 성능을 가진다.
